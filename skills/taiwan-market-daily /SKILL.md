# 台股每日市場研究 skill

## 概覽

本 skill 封裝了一套可重複執行的台股每日研究流程，由五個可組合模組組成。

**今日日期**：每次執行前先確認當天日期（`date +%Y年%m月%d日`），搜尋時帶入正確日期。

---

## 模組 A：大盤漲跌原因

**觸發詞**：「今日台股」「大盤為何漲/跌」「市場原因」

### 步驟
1. `web_search` 查詢：`台股大盤 今日漲跌原因 [今日日期]`
2. 補充搜尋：`費城半導體 美股 今日表現`（美股隔夜收盤是台股最重要的前因）
3. 從結果萃取以下維度並以**結構化摘要**輸出：
   - 指數表現（收盤點、漲跌點數、漲跌幅、成交值）
   - 主要驅動因素（上漲：列出 3～5 個；下跌：列出主要壓力）
   - 外部環境（費半、那斯達克、地緣政治、匯率）
   - 族群輪動（哪些類股強/弱）
   - 風險提示

### 輸出格式
用條列段落，不用表格。每個因素 1～2 句說明，最後一段是「潛在風險提醒」。

---

## 模組 B：供應鏈股票識別

**觸發詞**：「供應鏈股票」「相關概念股」「找出受惠廠商」

### 步驟
1. 從用戶提供的主題（或模組 A 萃取的類股資訊）確認要分析的產業主軸
2. 參照 `supply-chain-ticker-mapper` skill 的方法論執行：
   - 拆分供應鏈層次（台灣 → 美國 → 日本 → 韓國）
   - 每個層次列出 top 3～5 公司，附代號與受惠邏輯
3. 若主題涉及實體伺服器硬體，必須涵蓋：晶圓代工、先進封裝、ABF 載板、PCB/CCL、被動元件、記憶體、散熱、IC 設計、伺服器組裝
4. 以**分類 Markdown 表格**輸出（每類一個表：公司｜代號｜交易所｜受惠邏輯）

### 注意
- 不評估買賣，只做識別與分類
- 若識別結果超過 15 檔，詢問用戶是否要分批分析或聚焦特定類別

---

## 模組 C：即時報價儀表板

**觸發詞**：「查股價」「近期價格」「漲幅」「報價」

### 步驟
1. 從對話中收集目標股票代號（台股加 `.TW` 後綴）
2. 呼叫 `yfinance:get_batch_quotes` 一次取得所有報價
3. 計算每檔當日漲跌幅 = `(price - previous_close) / previous_close * 100`
4. 用 `visualize:show_widget` 渲染互動式卡片儀表板

### 儀表板卡片欄位
- 股票代號（等寬字型）、公司名稱
- 當日漲跌幅 badge（漲綠/跌紅）
- 當日收盤價
- 前收價 ＆ 漲跌點數
- 成交量（以萬/億顯示）

### 分組邏輯
按供應鏈類別分組（section header），與模組 B 一致。

---

## 模組 D：個股新聞分析

**觸發詞**：「新聞」「為何漲」「催化劑」「最新消息」

### 步驟
1. 優先從對話中確認要分析的標的（通常是當日大漲/大跌的個股）
2. 對每個標的：
   - 呼叫 `yfinance:get_news` 拉取最新新聞
   - 若 yfinance 新聞為空或過舊（>7 天），補充 `web_search`：`[公司名稱] [今日日期] 新聞`
3. 為每個標的整理：
   - 今日催化劑（1～3 個，附來源引用）
   - 基本面背景（近期財報亮點或法說重點，若有）
   - 潛在風險（短線過熱、消息未確認等）

### 輸出格式
每個標的一個 H3 小節，包含：漲幅、催化劑、風險。語氣客觀，不做買賣建議。

## 模組 E：大盤走勢 + 相關係數分析

**觸發詞**：「走勢圖」「大盤走勢」「相關係數」「關聯分析」「與大盤比較」

### 步驟

#### E1：大盤走勢圖
1. `yfinance:download_data` 下載 `^TWII`，`range=3mo`，`interval=1d`
2. 整理收盤價序列，用 `visualize:show_widget` + Chart.js 繪製：
   - 折線圖，Y 軸以「K」為單位
   - 標記關鍵轉折點（每段漲跌的起點/高點）
   - 上方 metric cards：今日收盤、今日漲跌、盤中高點、3 個月漲幅

#### E2：相關係數分析
1. 同時下載大盤 + 所有目標個股歷史數據（單次 `yfinance:download_data` 多 symbol）
2. 用 Python 計算（透過 `bash_tool`）：
   - Pearson 相關係數（以日報酬序列計算）
   - 3 個月累積報酬
   - 今日漲跌幅
   - 超額報酬（個股 3M 報酬 − 大盤 3M 報酬）
3. 正規化所有序列（基期=100），用 Chart.js 繪製多線折線圖
4. 相關係數卡片分類：高（r≥0.7）、中（0.5≤r<0.7）、低（r<0.5）

#### E3：解讀框架（每次執行後必須輸出）
- 高相關股：跟隨大盤節奏，beta 高，適合趨勢操作
- 低相關股：受個別事件驅動，需關注自身催化劑是否持續
- 超額報酬最高者：確認是否有獨立催化劑支撐，或只是追漲
- 超額報酬最低者：落後補漲空間評估

## 完整日報模式

用戶說「完整日報」「全部分析」「每日研究」時，依序執行：

```
A（大盤原因）→ B（供應鏈識別）→ C（報價儀表板）→ D（個股新聞）→ E（走勢+相關係數）
```

每個模組執行完後，詢問用戶是否繼續下一個，或直接全部跑完再呈現。
